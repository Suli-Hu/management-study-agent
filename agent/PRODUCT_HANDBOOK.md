## 1. 产品是什么

**Anytime Study（站点品牌 sususu）** 是一个 **多学科考研 / 大学院向** 的 **理论知识库 + 学习辅助** Web 应用。每个 **学科（discipline）** 有独立内容空间；用户在单一账号下可被授权访问一个或多个学科。

**核心价值**：把 **学派 → 学者 → 知识点（KP）** 连成可检索、可跳转的结构化笔记；支持 **中日双语正文**（按条目）；支持 **学习记录**（个人向）。

---

## 2. 定位与目标用户

| 维度 | 说明 |
|------|------|
| **场景** | 专业课复习、名词体系梳理、考前过知识点 |
| **设备** | 桌面 + **iPad**（小屏触控与 Safari 行为需认真对待） |
| **语言** | 主 **中文**；部分学科提供 **日文** 正文或标题；英文多作为 **术语括注** 或标题副线 |
| **角色** | **学习者**（读）、**编辑者/老师**（维护内容）、**管理员**（学科与用户） |

---

## 3. 内容模型（信息架构）

自上而下（4 层 + 视图）：

1. **Theme（学派组）** — 学科内对 **学派** 的分组（元数据，用于组织与导航语境）。  
2. **School（学派）** — 理论簇 / 卡片入口；一个学派下有多个 **KP**；页面 URL：`/{学科key}/{学派key}`。  
3. **Scholar（学者）** — 人物与贡献；可与多个 KP 关联；URL：`/{学科key}/scholars/{学者key}`。  
4. **KP（知识点）** — 最小阅读单元；结构化正文（多种 **format**：叙述 / 列表 / 手风琴 / 对比 / 四格等）；URL：`/{学科key}/kp/{kpId}`。  
5. **View（视图）** — **仅影响学科首页** 上学派卡片的 **分组与排序**；不改动 KP 本体。API-first 新学科可能暂无视图，系统可 **兜底列出全部学派**。

**标签（Tag）**：学科级 **标签库**（带颜色）；KP 可挂多个 tag key。与 **学派** 不同维：学派用于「从流派走进内容」，tag 用于「大类/模块/上色」。

**数据真源**：线上内容以 **Cloudflare D1** 为准；GitHub 承载 **应用代码**（不作为内容主库）。

## 4. 主要功能地图（用户可见）

### 4.1 顶部导航（与学科上下文相关）

- **学习记录**：个人学习日志、进度相关能力（含分享链接等能力，以线上为准）。  
- **理论**（或学科首页）：进入当前学科的 **学派/视图** 总览。  
- **学者**：学者列表与详情。  
- **知识点**：KP 列表或直达 KP 阅读流（以具体路由为准）。

### 4.2 学科首页 `/{discipline}`

- **视图切换**（tab/chip）：切换不同 **View** 下的分组与说明。  
- **学派卡片**：点击进入学派页。  
- 依赖 D1 中 `view` 表；若无视图可有 **兜底学派列表**。

### 4.3 学派页 `/{discipline}/{school}`

- 该学派下的 **KP 列表**、分栏阅读（左列表右详情等交互以当前实现为准）。  
- 编辑者可 **改序**、进编辑入口（权限受控）。

### 4.4 学者页 `/{discipline}/scholars/{key}`

- 简介、贡献、关联学派、关联 KP 等 tab。

### 4.5 KP 详情页 `/{discipline}/kp/{id}`

- **标题**（中/日/英副标题按字段展示）。  
- **元信息**：关联 **学者**（可点）、**年份**（自由文本，常显式为 `'77` 等缩写）。  
- **Chip 行**：**语言切换**（中↔日，需有日文正文）、**学派**（可点，可多枚）、**标签**（当前多为不可点，色来自标签库）。  
- **正文**：结构化渲染；可有 **评价模块**（义/限/例等字段，依学科与条目）。  
- **悬浮语言球**：与语言切换同源（移动端强化）。

### 4.6 编辑与后台（需权限）

- **KP / 学派 / 学者 / 视图 / 学派组** 等编辑器（浏览器内）。  
- **学科管理**、**用户与租户成员**、**API Token** 等（管理员向）。  
- 具体能力与路由以线上版本为准；设计稿可参考 `agent/design/` 内 PRD。

### 4.7 搜索与其他

- **搜索**（若开启）：跨 KP/学者/学派等（权限与公开范围依配置）。  
- **设计规范镜像页**：https://study.sususu.org/design-system

## 5. 权限与多租户（简述）

- 按 **学科** 隔离内容；用户通过 **tenant_member** 获得某学科的 **owner / editor / viewer**。  
- **Super-admin**（配置项）可跨学科。  
- **API Token** 可收窄到指定学科 scope。  
- 设计师需知：**同一 UI 在不同角色下** 会显示/隐藏 **编辑入口、拖拽、管理控件**。

---

## 6. 设计与交付约束（给设计师）

- **设计系统**：线上 `design-system` 为 **L1/L2/L3 色阶、语义色、圆角、阴影、触控最小尺寸** 的参考；大改前先对齐，避免另起一套视觉。  
- **触控**：语言切换、重要操作宜满足 **约 44×44pt** 级命中区（见设计系统说明）。  
- **变体与扩展**：Chip 区需考虑 **多学派 + 多标签 + 语言控件** 并存；避免「可点与不可点」仅靠颜色区分。  
- **工作流**：前端可见改动默认 **本地起服务 + 可交互 demo**（见 `FRONTEND_WORKFLOW.md`）。

---

## 7. 内容与本地化（给老师/录入，与设计相关）

- **中日分字段**：`title.zh` / `body.zh` vs `title.ja` / `body.ja`，勿在同一语言字段内混排。  
- **英文**：术语多用 **中文（English）** 括注；详见 `TEACHER.md`「Multilingual content rules」。  
- **标签**：使用学科标签库 **key**，非自由滥造。
